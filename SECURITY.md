# Security Policy

## Philosophie
La sécurité n'est pas une option, c'est le socle de toute infrastructure. En tant que professionnel passionné et méticuleux, j'applique les principes du moindre privilège, de la défense en profondeur et du *Security by Design* sur l'ensemble de mes architectures. 

Ce dépôt est un laboratoire de démonstration (OPSEC) conçu pour partager publiquement des pratiques robustes. Bien que les configurations ici soient des simulations (pas mon infrastructure de production réelle), elles sont écrites pour être prêtes à l'emploi et sécurisées par défaut.

## Versions Supportées
Actuellement, seule la branche principale (`main`) est activement maintenue.

| Version | Supportée          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |

## Signaler une Vulnérabilité
Si vous découvrez une faille de sécurité, une mauvaise pratique potentielle ou une fuite accidentelle dans l'une des configurations de ce dépôt, je vous remercie par avance de m'en informer en privé afin de ne pas exposer ceux qui réutiliseraient ce code.

**Veuillez ne pas créer d'Issue publique pour les problèmes de sécurité.**

Envoyez-moi un message direct ou contactez-moi via les réseaux professionnels liés à ce profil GitHub.
